MySQL

MySQL分为server层和存储引擎层

  • server层负责建立连接，分析和执行SQL
  • 存储引擎层负责数据的存储和提取

连接过程

连接命令

mysql -h $ip -u $user -p

连接过程需要经过TCP三次握手
完成TCP连接建立后连接器开始验证用户名和密码

查看MySQL被多少个客户端连接了

show processlist

MySQL定义了空闲连接的最大空闲时长，如果空闲连接潮沟这个时间，连接器会自动断开
也可以用kill connection +id 主动断开

MySQL的最大连接数由max_connections控制

有短连接和长连接的概念，短连接执行一次sql之后断开，长连接可以多次执行SQL语句

解决长连接占用内存问题

  • 定期断开
  • 客户端主动重置

查询缓存

如果SQL被解析出是查询语句，就会在缓存中寻找缓存数据，查询缓存是以key- value形式保存在内存中

MySQL8.0删除了查询缓存阶段，在之前的版本中，可以控制参数来关闭查询缓存

解析SQL

解析器主要做两件事

  • 词法分析，将字符串识别出关键字出来
  • 语法分析，判断SQL语句是否满足MySQL语法，并且构建出SQL语法树，如果语法错误，就会报错

执行SQL

对于select语句，有下面三个阶段

  • prepare 阶段，预处理阶段
    • 检查SQL查询语句中的表或字段是否存在
    • 将 * 扩展为表上所有的列
  • 优化器
    • 将 SQL 查询语句的执行方案确定下来
    • 有一些查询既可以使用主键索引，也可以使用普通索引，需要优化器决定使用哪个索引
  • 执行器
    • 执行器就会和存储引擎交互了，交互是以记录为单位的。
    • 三种执行方式
    • 主键索引查询
    • 全表扫描
    • 索引下推

MySQL的记录如何存储

表空间文件的结构

  • 行（row）：记录都是按照row存放
  • 页（page）: InnoDB 的数据是按「页」为单位来读写的
  • 区（extent）
  • 段（segement）

每个数据库表的行格式都有「NULL 值列表」吗？

当数据表的字段都定义成 NOT NULL 的时候，这时候表里的行格式就不会有 NULL 值列表了。

行溢出后，MySQL 是怎么处理的？

将溢出页的地址存储在原本的数据里（20字节），然后在溢出页存储剩余数据

索引的分类

  • 按「数据结构」分类：B+tree索引、Hash索引、Full-text索引。
  • 按「物理存储」分类：聚簇索引（主键索引）、二级索引（辅助索引）。
  • 按「字段特性」分类：主键索引、唯一索引、普通索引、前缀索引。
  • 按「字段个数」分类：单列索引、联合索引。
